# FingerprintReg — Fingerprint 账号全自动批量注册机

一键批量注册账号，独立指纹浏览器 + 全流程自动化 + Web 控制面板。支持全自动验证码（打码平台 + AI 识别），也支持手动过验证码。

---

## 功能亮点

- **全自动验证码** — 三级自动化：打码平台（YesCaptcha/CapSolver）→ AI 视觉识别（Gemini）→ 手动兜底
- **独立指纹浏览器** — 每个窗口独立指纹（平台/品牌/时区/CPU），互不关联
- **自动申请临时邮箱** — 基于 YYDS Mail API，无需手动注册邮箱
- **自动点击 "I'm not a robot"** — reCAPTCHA 复选框自动点击
- **自动接收验证码并填入** — 6 位 OTP 逐位精准键入
- **自动填写密码和姓名** — 随机真人英文名，密码统一配置
- **双模式国家选择** — 自选模式（搜索 + 随机）/ 常用模式（收藏夹标签快选）
- **一键填卡** — 注册完直接弹出信用卡填写界面，支持智能识别粘贴
- **一键登录 + 检测绑卡** — 浏览器关闭后重新登录，自动检测绑卡状态
- **导入账号** — 粘贴已有账号批量导入，支持多种格式自动解析
- **国家自动识别** — 注册时自动记录，已有账号可二次检测补全
- **绑卡状态持久化** — 已绑/未绑状态保存到文件，重启不丢失，后台实时监测变化
- **多窗口并行** — 注册、登录、填卡全部支持并发，效率翻倍
- **自动选国家 / 自动点绑卡** — 可独立开关控制
- **Web 控制面板** — 实时进度、4 种主题、延迟参数可调、表单数据持久化
- **注册结果永久保存** — 账号信息自动保存到 JSON + CSV，不会丢失

## 快速开始

**1.** 解压下载的 ZIP 文件

**2.** 打开解压后的文件夹，在地址栏输入 `cmd` 回车，打开命令行

**3.** 安装依赖（只需要第一次）：
```bash
pip install -r requirements.txt
```

**4.** 启动程序：
```bash
python -m jetbrainsreg
```

**5.** 浏览器自动打开 Web 控制面板 → **首次使用先在顶部填入 YYDS Mail API Key 并保存** → 设置密码和窗口数 → 点「开始注册」

**6.** 浏览器窗口自动打开 → 自动填邮箱 → 自动点 reCAPTCHA → **全自动过验证码**（或手动兜底）→ 后续全自动完成！

详细步骤请看压缩包里的 **使用教程.txt**。

## 注册流程（8步）

```
步骤 1  自动申请临时邮箱（YYDS Mail API）
  ↓
步骤 2  打开注册页 → 自动填入邮箱 → 点击 Continue
  ↓
步骤 3  自动点击 reCAPTCHA "I'm not a robot" 复选框
  ↓
步骤 4  全自动验证码：打码平台 → AI 视觉识别 → 手动兜底（可选）
  ↓
步骤 5  自动检测验证完成 → 自动提交 → 自动接收邮件验证码 → 自动填入 OTP
  ↓
步骤 6  自动填写姓名和密码 → 勾选协议 → 点击 Create account
  ↓
步骤 7  自动检测注册结果
  ↓
步骤 8  自动跳转 tokens 页 → 自动选国家 → 自动弹出 Add credit card（可关闭）
```

### 全自动验证码说明

勾选「全自动验证码」后，验证码处理按以下优先级自动降级：

1. **打码平台**（YesCaptcha / CapSolver）— 通过 API 提交 reCAPTCHA sitekey，获取 token 直接注入页面，成功率最高
2. **AI 视觉识别**（Gemini 3 Flash）— CDP 截图 + AI 返回坐标点击，免费内置，无需配置
3. **手动兜底** — 以上都失败时，等待用户手动完成（无时间限制）

不勾选则为纯手动模式（和之前一样）。

## 指纹浏览器（可选）

使用指纹浏览器可以让每个注册窗口拥有独立的浏览器指纹，降低关联风险。

放置位置（脚本自动搜索）：
```
FingerprintReg/
├── Chromium/
│   └── Application/
│       └── chrome.exe        ← 指纹浏览器放这里
├── jetbrainsreg/
├── requirements.txt
└── ...
```

也可以在控制面板中选择使用系统 Chrome 或 Edge（无需指纹浏览器）。

## 注意事项

- 密码建议使用**英文字母和数字**组合，避免特殊符号导致填写异常
- 注册成功后浏览器窗口**保持打开不关闭**，方便后续填卡操作
- CMD 窗口是后台服务，注册过程中**不要关闭**
- 如果验证码长时间加载不出来，检查**梯子是否正常**
- 每个浏览器窗口使用**独立指纹和独立数据目录**，互不影响
- 注册结果保存在 `output/` 文件夹，`accounts.csv` 可以用 Excel 打开
- 右上角齿轮按钮可以调整主题和延迟参数
- YYDS Mail API Key 保存在 `output/settings.json`，只需配置一次

## 常见问题

**Q: 提示 "YYDS Mail API Key 未配置"？**
> 访问 https://vip.215.im/api-keys 注册并创建 API Key，然后在控制面板顶部粘贴保存。Key 以 `AC-` 开头。

**Q: 提示找不到 chrome.exe？**
> 在控制面板的浏览器选项中选择 Chrome 或 Edge（使用系统浏览器），或将指纹浏览器放到 `Chromium/Application/` 目录下。

**Q: reCAPTCHA 一直加载不出来？**
> 检查代理/VPN 是否正常。Google reCAPTCHA 在中国大陆无法直接访问。

**Q: 验证邮件一直收不到？**
> YYDS Mail 在 Cloudflare 上，需要**走梯子代理**。如果频繁超时，检查代理是否正常。

**Q: 注册成功但密码不对？**
> 检查控制面板中的「统一密码」字段。密码在所有账号间共享。

**Q: 一键登录后绑卡状态全部失败？**
> 浏览器关闭后旧的检测方式失效。使用「一键登录 + 检测」功能会自动打开新浏览器登录并检测。

**Q: 可以同时开几个窗口？**
> 控制面板中设置窗口数，最多 10 个。建议根据电脑性能选择 3-5 个。
